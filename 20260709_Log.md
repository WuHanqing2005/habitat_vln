**quat_to_rotation_vector 兼容性修复**（第 82-114 行）：
habitat-sim 0.3.3 没有 `quat_to_rotation_vector()` 函数，手动实现四元数→前向向量转换：
```python
@staticmethod
def _quat_to_heading(rotation) -> float:
    try:
        import habitat_sim.utils.common as utils
        try:
            forward = utils.quat_to_rotation_vector(rotation)
            return math.degrees(math.atan2(forward[0], forward[2]))
        except AttributeError:
            pass
        # 手动计算
        qx, qy, qz, qw = rotation.x, rotation.y, rotation.z, rotation.w
        fx = 2.0 * (qx * qz + qw * qy)
        fz = 2.0 * (qy * qz - qw * qx)
        return math.degrees(math.atan2(fx, fz))
    except Exception:
        return 0.0
```

**RGBA→RGB 转换**（第 351-353 行）：
Habitat-Sim 返回的 RGB 传感器数据是 4 通道 RGBA，需要转换为 3 通道 RGB：
```python
if rgb.shape[2] == 4:
    rgb = rgb[:, :, :3]
```

#### 3.3 实现 navigation_loop.py — 导航循环

**文件**: `vln_agent/navigation_loop.py` (265 行)

**类**: `NavigationLoop`

核心循环逻辑（感知-决策-行动）：

```
for step in range(1, max_steps + 1):
    # 1. 感知阶段
    image = agent.capture_image()
    agent.save_step_image(image, step)     # 保存图片到 Pictures/
    agent.write_video_frame(image)         # 写入视频帧
    
    # 2. 决策阶段
    if is_stuck:
        forced_action = agent.handle_stuck()
    else:
        ai_result = agent.vision_client.analyze_navigation_step(image, ...)
    
    # 3. 执行阶段
    if action == "arrived":
        agent.state["arrived"] = True
        break
    execution_result = agent.execute_action(action)
    
    # 4. 记录
    agent.logger.log_step(...)
    agent.trajectory_data.append({...})
```

**防循环机制**：
- 卡住检测：连续 N 步位置未变化视为卡住

### 问题修复记录

#### 问题 1：quat_to_rotation_vector 不存在

**症状**：habitat-sim 0.3.3 中没有 `quat_to_rotation_vector()` 函数，导致转向动作后朝向无法更新，智能体卡在循环中。

**修复**：在 `agent.py` 中实现 `_quat_to_heading()` 静态方法，优先尝试 API 调用，失败则手动计算四元数→前向向量。

**涉及文件**：`vln_agent/agent.py` 第 82-114 行

#### 问题 2：numpy 2.0.0 与 habitat-sim 0.3.3 不兼容

**症状**：创建仿真器时崩溃：
```
TypeError: ufunc 'exp' not supported for the input types
```

**原因**：habitat-sim 0.3.3 依赖 `numpy-quaternion` 库，该库的 `np.exp(quats)` ufunc 在 numpy 2.x 中无法注册。

**修复**：将 numpy 从 2.0.0 降级到 1.26.4：
```bash
conda install -n habitat numpy=1.26.4 -y
```

#### 问题 3：RGBA 4通道图像

**症状**：Habitat-Sim 返回的 RGB 传感器数据是 4 通道 RGBA，但下游代码期望 3 通道 RGB。

**修复**：在 `capture_image()` 中自动检测并转换：
```python
if rgb.shape[2] == 4:
    rgb = rgb[:, :, :3]
```
